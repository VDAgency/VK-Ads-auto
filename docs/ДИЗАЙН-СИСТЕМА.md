# ДИЗАЙН-СИСТЕМА.md — мастер-инструкция дизайн-конвейера

> **Назначение.** Это не дизайн конкретного проекта, а **мастер-правило**, из которого Claude формирует `DESIGN.md` под каждый отдельный проект. Инструменты ниже ставятся **per-project** (в папку проекта), а не глобально — фронтенд нужен не во всех проектах. Этот файл — справочник и чек-лист: что входит в дизайн-конвейер, как он работает и как из него родить `DESIGN.md`.
>
> **Зачем сейчас.** Нужен для Блока 2 проекта VK Ads (клиентский веб-кабинет), чтобы результат выглядел профессионально и клиент захотел продолжить сотрудничество. Дальше — переиспользуется на любых сайтах/лендингах.

---

## 1. Философия (главное)

1. **«Framer-качество» — это связка, а не один генератор.** Ни один опен-сорс не даёт Framer «в один клик». Качество даёт: готовые анимированные блоки + дизайн-система + вкус + визуальная петля.
2. **Сначала готовый блок, потом кастомизация.** Не генерим компонент с нуля — берём проверенный анимированный блок из реестра и подгоняем под `DESIGN.md`. Это быстрее и консистентнее.
3. **Дизайн-система решается один раз.** Шрифты, палитра, ритм, motion фиксируются в `DESIGN.md` проекта и переиспользуются на всех экранах — чтобы не угадывать заново и не править по кругу.
4. **Визуальная петля вместо повторов.** Агент рендерит → делает скриншот → сравнивает с направлением → правит сам. Это лекарство от «правь-правь по сто раз».

## 2. Состав конвейера (роли инструментов)

| Слой | Инструмент | Роль | Тип | Ключ |
|---|---|---|---|---|
| Вкус и критика | **Impeccable** | направление, анти-слоп, руководство по motion, команды `/audit` `/polish` `/animate` `/critique` `/quieter` | skill | нет |
| Линтер дизайна | **Impeccable CLI** | `npx impeccable detect` — детерминированно ловит «AI-слоп» без LLM | CLI | нет |
| Механизм компонентов | **shadcn MCP** | живой реестр shadcn + установка блоков из любых реестров | MCP | нет |
| Готовые анимированные блоки | **Aceternity UI** (200+), **Magic UI** (150+), **react-bits** (110+, часть CSS/RSC) | «Framer-ощущение»: hero, bento, параллакс, text-reveal, спотлайты | реестры (shadcn CLI) | нет (free-ядро) |
| Код анимаций | **Motion** (бывш. Framer Motion) / **GSAP** | физически двигает элементы | npm-пакет | нет |
| Визуальная проверка | **Playwright MCP** | скриншот своего рендера → сравнение → автоправка (и он же под автоматизацию kotbot) | MCP | нет |
| Генератор макетов (опц.) | **Superdesign** | макеты/вайрфреймы из текста в IDE | skill/расширение | нет |
| Шрифты | **кириллический шрифт-набор** | обязательно для RU-интерфейса | skill/ассеты | нет |

Принцип: весь набор **без внешних ключей** — это сознательно, чтобы обойти проблему VPN/маршрутизации ключей. 21st.dev Magic (платный генератор) — не берём; его заменяют бесплатные реестры выше.

## 3. Пайплайн сборки экрана/страницы

1. **Shape** — спланировать UX/UI до кода (`/impeccable shape`): структура, иерархия, что на экране делает работу.
2. **Подобрать готовый блок** из реестра (Aceternity/Magic UI/react-bits) под задачу — не генерить с нуля. Установка через shadcn CLI.
3. **Кастомизировать под `DESIGN.md`** — подставить шрифты, палитру, спейсинг проекта (токены в CSS-переменных).
4. **Animate** — осмысленное движение (`/impeccable animate` + Motion): page-load последовательность, scroll-reveal, hover. Обязательно `prefers-reduced-motion`.
5. **Audit** — `/impeccable audit` + `npx impeccable detect` (без ключа): анти-слоп, доступность, перф, тач-таргеты.
6. **Визуальная проверка** — Playwright: скриншот → сравнить с направлением `DESIGN.md` → правка. Повторять до соответствия.
7. **Polish** — `/impeccable polish`: финальная согласованность с дизайн-системой, готовность к выкату.

## 4. Анти-паттерны (запреты — из Impeccable)

- ❌ Дефолтные шрифты (Inter, Roboto, Arial, системные).
- ❌ Фиолетово-синие градиенты на белом.
- ❌ Карточки внутри карточек; «три скруглённые карточки» в hero.
- ❌ Чистый чёрный/серый — нейтрали всегда тонировать.
- ❌ Серый текст на цветном фоне.
- ❌ Bounce/elastic easing (выглядит устаревшим).
- ❌ Россыпь микроанимаций вместо одной оркестрованной.
- ✅ **Кириллица обязательна** — выбранные шрифты должны поддерживать русский (трендовые часто не поддерживают).
- ✅ 8px-ритм спейсинга; одна доминанта + один резкий акцент; все цвета в CSS-переменных.

## 5. Регистр «бренд vs продукт» (важно для тона)

- **Бренд (лендинги, маркетинг):** можно смелее — выразительная типографика, анимации, вау-блоки Aceternity. Команды `/bolder`, `/overdrive`, `/delight`.
- **Продукт (внутренние кабинеты, порталы):** сдержанность, чистота, консистентность, точечная анимация. Команды `/quieter`, `/distill`, `/polish`.

**Для проекта VK Ads, Блок 2 (клиентский мини-кабинет) — это ПРОДУКТ.** Цель: чистый, профессиональный, понятный B2B-портал. Анимации точечные (плавные переходы, аккуратные состояния загрузки), без флеша. Вау-эффекты Aceternity приберечь для будущего маркетингового лендинга.

## 6. Как из этого файла родить `DESIGN.md` под проект

При старте фронт-части Claude генерит `DESIGN.md` проекта со следующими блоками (Impeccable `/teach` помогает собрать):

1. **Тип и тон** — бренд или продукт; одно концептуальное направление (одной фразой).
2. **Шрифты** — display / body / mono, **обязательно с кириллицей**; какие шрифты банить.
3. **Палитра** — одна доминанта + резкий акцент, в CSS-переменных (желательно OKLCH); тёмная тема.
4. **Спейсинг и сетка** — 8px-ритм; принципы сеток.
5. **Motion** — easing-кривые, длительности, где уместна анимация, обязательный `reduced-motion`.
6. **Источники компонентов** — какие реестры подключены (Aceternity/Magic UI/react-bits) и правило «сначала блок → потом кастом».
7. **Доступность и адаптив** — mobile-first, контраст, фокус-состояния.
8. **Визуальный чек** — после изменений UI обязателен скриншот через Playwright и сравнение с направлением.

### Шаблон `DESIGN.md` (рыба)
```markdown
# DESIGN.md — <проект>
## Тон: <бренд|продукт> · Направление: <одна фраза>
## Шрифты: display=<...> body=<...> mono=<...> (кириллица: да); бан: Inter, Roboto, системные
## Палитра: доминанта=<...> акцент=<...> (CSS-переменные/OKLCH); тёмная тема: <да|нет>
## Спейсинг: 8px-ритм · сетка: <...>
## Motion: easing=<...> длительности=<...> где=<page-load|scroll|hover> · reduced-motion: обязательно
## Компоненты: реестры=<Aceternity|Magic UI|react-bits> · правило: сначала блок → потом кастом
## Доступность/адаптив: mobile-first, контраст AA, фокус-состояния
## Визуальный чек: Playwright screenshot → сравнить с направлением → править
```

## 7. Установка под проект (когда берёшься за фронт; НЕ глобально)

```bash
# дизайн-вкус (skill, в проект)
cp -r <impeccable>/dist/claude-code/.claude <project>/

# механизм компонентов
claude mcp add shadcn -- npx shadcn@latest mcp
npx shadcn@latest init            # инициализация проекта shadcn

# готовые анимированные блоки (через shadcn CLI, по мере надобности)
npx shadcn add @magicui/<component>
npx shadcn add @aceternity/<component>

# анимации (код)
npm i motion        # бывш. framer-motion

# визуальная проверка (и под kotbot)
claude mcp add playwright -- npx @playwright/mcp@latest
npm i -D @playwright/test && npx playwright install

# линтер дизайна без ключа
npx impeccable detect src/
```

> Шрифты: подключить кириллические через next/font (Google Fonts) и зафиксировать в `DESIGN.md`. Никаких ключей.
