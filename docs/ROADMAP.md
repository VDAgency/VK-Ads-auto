# ROADMAP.md — План работ

> **Правила работы с этим файлом:**
> 1. Каждый выполненный пункт **сразу** отмечается чекбоксом `[x]`, чтобы ROADMAP всегда отражал реальное состояние.
> 2. Внутри фазы — подзадачи (строки). Фаза считается закрытой только после прохождения **Гейта качества** (см. ниже).
> 3. Порядок фаз менять можно, но контрольные точки Блоков (приёмка) — жёсткие.

Связанные документы: `PROJECT.md`, `BRIEF_SPEC.md`, `DECISIONS.md`.

---

## Гейт качества (выполняется в конце КАЖДОЙ фазы)

После завершения всех подзадач фазы прогнать полный набор проверок. **Коммит и пуш — только когда всё зелёное.**

- [ ] `ruff check .` — линт без ошибок
- [ ] `ruff format --check .` — форматирование
- [ ] `mypy .` — типизация без ошибок
- [ ] `pytest -q` — все тесты зелёные
- [ ] (Блок 2 / фронт) `eslint`, `prettier --check`, `tsc --noEmit`
- [ ] Все найденные ошибки исправлены, проверки прогнаны повторно
- [ ] Обновлены `ROADMAP.md` (чекбоксы) и при необходимости `DECISIONS.md`
- [ ] `git add -A && git commit -m "feat(phase-N): ..."` → `git push`
- [ ] Push прошёл CI и **авто-задеплоился на сервер** (см. Фаза 0); проверен healthcheck на сервере

> Принцип: красные проверки → не коммитим. Любой push в `main` = деплой на сервер. Поэтому в `main` попадает только зелёный код.

## Фаза 0 — Инфраструктура, репозиторий и авто-деплой

Цель: чтобы **сервер существовал с первого дня**, а каждый push автоматически выкатывался на него.

- [ ] Создать **публичный** GitHub-репозиторий проекта
- [ ] Структура проекта (`core/`, `bot/`, `services/`, `integrations/`, `db/`, `web/`, `tests/`, `infra/`)
- [ ] Определить контракт внутреннего API ядра (версионированный, `/api/v1/...`) — единый для всех фронтов (headless-ядро); бизнес-логика только в `core/`/`services/`
- [ ] Заложить швы мульти-тенанта: `account_id` во всех таблицах, скоупинг запросов по тенанту, конфиг интеграций (VK/бот/Google/Senler) — per-account, а не глобальные константы
- [ ] `pyproject.toml` с зависимостями; настроить `ruff`, `mypy`, `pytest`
- [ ] `docker-compose.yml`: FastAPI, PostgreSQL, Redis, n8n, бот
- [ ] `.env.example` (без секретов); реальные секреты — только в окружении сервера и в GitHub Actions Secrets
- [ ] Поднять Beget VPS, установить Docker + Docker Compose
- [ ] Настроить GitHub Actions: на push в `main` → прогон Гейта качества → SSH-деплой на Beget (`git pull` / `docker compose pull && up -d`)
- [ ] Все ключи для авто-деплоя (SSH-ключ, пароли БД, токены) — в **GitHub Actions Secrets** и в `.env` на сервере; **в публичный код секреты не коммитим**
- [ ] Healthcheck-эндпоинт `/health` в FastAPI; проверка после деплоя
- [ ] Базовый CI-тест (smoke) зелёный

> **Безопасность публичного репозитория:** код открытый, секреты — нет. Всё чувствительное (client_secret VK, токен бота, доступы Google/Senler, SSH) живёт в Actions Secrets и серверном `.env`, в репозиторий не попадает. `.gitignore` закрывает `.env`, ключи, креды.

**→ Гейт качества → commit → push → деплой.**
